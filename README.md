# Git과 함께하는 Python 첫 발자국

## 프로젝트 개요

이 프로젝트는 Python 기본 문법과 Git 기초를 함께 익히기 위한 콘솔 퀴즈 게임입니다.
사용자는 메뉴를 통해 퀴즈를 풀고, 새 퀴즈를 추가하고, 등록된 문제를 확인하고, 최고 점수를 관리할 수 있습니다.
데이터는 프로젝트 루트의 `state.json` 파일에 UTF-8 형식으로 저장되어 프로그램을 다시 실행해도 유지됩니다.
또한 콘솔 화면에서 상태에 맞는 이모티콘 아이콘과 컬러를 사용해 메뉴, 경고, 정답/오답, 힌트, 점수 메시지를 더 직관적으로 구분하도록 구성했습니다.

## 퀴즈 주제와 선정 이유

퀴즈 주제는 **Python과 Git 입문**입니다.
이번 미션의 핵심 학습 주제와 직접 연결되어 있어, 프로그램을 만들면서 동시에 관련 개념도 함께 복습할 수 있도록 구성했습니다.

## 과제 목표

### Python 기초

- 변수: 데이터를 이름으로 저장해 반복해서 사용하고, 프로그램 흐름에 따라 값을 바꿀 수 있다는 점을 이해합니다.
- 자료형: `int`, `str`, `bool`, `list`, `dict`의 역할과 차이를 구분하고 상황에 맞게 사용할 수 있습니다.
- 조건문: `if`, `elif`, `else`를 사용해 입력값이나 상태에 따라 다른 동작을 수행할 수 있습니다.
- 반복문: `for`와 `while`의 차이를 이해하고, 문제 순회나 재입력 처리에 적절하게 활용할 수 있습니다.
- 함수: 기능을 나누어 작성하고, 매개변수와 반환값을 사용해 코드를 재사용할 수 있습니다.

### 클래스와 객체

- 클래스: 관련된 데이터와 기능을 하나의 구조로 묶어 관리하는 이유를 이해합니다.
- `__init__`와 `self`: 객체가 생성될 때 초기값을 설정하고, 각 객체 자신의 속성에 접근하는 방식을 설명할 수 있습니다.
- 속성과 메서드: 문제 데이터, 선택지, 정답, 게임 진행 같은 역할을 속성과 메서드로 분리해 표현할 수 있습니다.

### 파일 입출력

- 파일 읽기/쓰기: 프로그램 실행 결과를 파일에 저장하고 다시 불러오는 기본 흐름을 이해합니다.
- JSON: 구조화된 데이터를 저장하기 좋은 형식으로서 `state.json`의 역할을 설명할 수 있습니다.
- 예외 처리: `try/except`를 사용해 파일이 없거나 손상된 경우에도 프로그램이 안전하게 동작하도록 만들 수 있습니다.

### Git 기초

- Git의 역할: 변경 이력을 기록하고, 기능 단위로 작업을 나누며, 협업 준비가 가능한 이유를 이해합니다.
- 기본 명령어: `init`, `add`, `commit`, `push`, `pull`, `checkout`, `clone`이 각각 무엇을 하는지 설명할 수 있습니다.
- 브랜치 활용: 기능을 별도 브랜치에서 구현한 뒤 `merge`로 통합하는 흐름을 경험할 수 있습니다.

## 추천 Git 작업 순서

아래 순서대로 진행하면 미션 요구사항의 커밋 흐름을 자연스럽게 만들 수 있습니다.

1. `git add .`
2. `git commit -m "Init: 프로젝트 기본 파일 생성"`

8. `git merge feature/play-quiz`
9. `git commit -m "Feat: 퀴즈 추가 기능 구현"`
10. `git commit -m "Feat: 퀴즈 목록 기능 구현"`

13. `git commit -m "Docs: README 문서화"`

## clone / pull 실습 예시

1. GitHub에 저장소를 `push`합니다.
2. 다른 디렉터리에서 저장소를 복제합니다.

```bash
git clone <GitHub 저장소 URL>
```

3. 복제한 디렉터리에서 README에 한 줄을 추가한 뒤 커밋하고 `push`합니다.
4. 원래 작업 디렉터리로 돌아와 아래 명령어로 변경 사항을 가져옵니다.

```bash
git pull
```

실습 메모: GitHub에서 `clone`한 복제본으로 README를 수정한 뒤 `push`했고, 원래 작업 디렉터리에서 `pull`로 반영을 확인했습니다.
